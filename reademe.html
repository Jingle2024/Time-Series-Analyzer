Core Distinctions
⏱
Temporal Accumulation
Operates along the time axis. Collapses fine-grained observations into coarser time buckets. A daily series becomes weekly by summing (for flow quantities like sales) or averaging (for stock quantities like inventory). Preserves total mass or average level as appropriate.
y_W = Σ y_d for d ∈ week (flow)
y_W = avg(y_d) for d ∈ week (stock)
🌲
Hierarchy Aggregation
Operates across series (cross-sectional). Leaf series (SKU, store, region) are aggregated upward through a tree structure. Parent nodes are derived from children using sum, weighted mean, or proportional allocation. Ensures top-down coherence.
y_parent,t = Σ y_child_i,t (additive)
y_parent,t = Σ w_i · y_child_i,t (weighted)
🔀
Combined Operations
Both operations are orthogonal and can be applied together. First accumulate each leaf series to the desired interval, then aggregate across the hierarchy. Order matters: accumulate first ensures temporal consistency before spatial aggregation.
Step 1: y_leaf,W = accum(y_leaf,d)
Step 2: y_total,W = Σ y_leaf,W
📉
Decomposition
Separates a time series into interpretable components. Additive model assumes components add up; multiplicative model assumes they multiply. STL (Seasonal-Trend decomposition using LOESS) is robust to outliers and handles multiple seasonal periods.
Additive: y_t = T_t + S_t + C_t + R_t
Multiplicative: y_t = T_t × S_t × C_t × R_t
💤
Intermittency (ADI/CV²)
ADI = average interval between non-zero demands. CV² = variance of non-zero demand size. Series with ADI > 1.32 and CV² < 0.49 are Intermittent. High ADI + high CV² = Lumpy. Each class needs different forecasting treatment.
ADI = n_periods / n_nonzero
CV² = (σ_nz / μ_nz)²
📡
Interval Recommendation
The system scores candidate intervals by: (1) dominant spectral frequency alignment, (2) signal-to-noise ratio after resampling, (3) autocorrelation decay rate, and (4) minimum periods needed for reliable decomposition (≥2 full cycles).
score(f) = SNR(f) × ACF_score(f) × coverage(f)
→ top-3 intervals recommended

Detailed Data Flows
⏱ Temporal Accumulation Flow
1
Receive raw series — irregular or regular timestamps, any granularity
2
Validate timestamp index — parse timezone, detect DST transitions, sort
3
Detect quantity type — flow (sales, events) vs stock (inventory, price) → determines method
4
Resample to target interval — e.g., hourly → daily → weekly → monthly
5
Handle boundaries — partial periods at start/end, daylight savings gaps
6
Produce comparison table — show accumulated values at multiple intervals side by side
7
Compute information loss metric — quantify signal lost at each coarser interval
🌲 Hierarchy Aggregation Flow
1
Parse hierarchy definition — from config file (JSON/YAML) or inferred from column names
2
Identify leaf series — lowest-level nodes with no children
3
Align timestamps — ensure all leaves cover the same time range (impute if needed)
4
Aggregate bottom-up — compute each level from its children using the specified method
5
Validate coherence — verify parent_t = Σ children_t at every timestep
6
Level-effect report — show how variance, trend, seasonality change per aggregation level
7
Export all levels — structured output with level labels for downstream use
🔍 Analysis Agent Flow
1
Stationarity test — ADF, KPSS, PP test. Determine differencing order.
2
Trend extraction — LOESS, HP filter, linear regression, structural break detection (Chow test)
3
Seasonal detection — ACF/PACF plots, periodogram peaks, STL seasonal component
4
Cycle isolation — Hodrick-Prescott or Baxter-King bandpass filter for business cycles
5
Residual analysis — Ljung-Box test, normality test, heteroskedasticity check (ARCH)
6
Component strengths — Fs (seasonal strength) and Ft (trend strength) metrics (0–1)
⚙️ Data Preparation Flow
1
Apply transforms — log(1+y), Box-Cox (λ optimized), first/seasonal differencing
2
Scale / normalize — MinMax, StandardScaler, RobustScaler (for outlier-heavy data)
3
Create lag features — y_t-1, y_t-7, y_t-30, y_t-364 based on ACF significant lags
4
Rolling features — rolling mean, std, min, max over 7/14/28-period windows
5
Calendar encoding — month, day-of-week, week-of-year, is_holiday (Fourier or one-hot)
6
Train/Val/Test split — temporal split with configurable horizon. No random shuffle.

Agent Communication Model
Message-Passing Architecture
All agents communicate via a shared Context Store. The Orchestrator uses a ReAct loop to plan, dispatch, and observe. Agents are stateless — they read from and write to the store, enabling parallel execution and transparent debugging.
